# Blog Tags Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a predefined tag system to the Writing section — tags managed as a pagescms.org collection, multi-tag support per post, "POSTS BY TAGS" sidebar section, "Show More" modal, tag-filtered sidebar view, and a blog home landing at `/writing`.

**Architecture:** Tags live in `src/content/writing/tags/*.json` (one per tag), loaded at build time via `import.meta.glob` (same pattern as categories). `Writing.tsx` gains three new local state slices (`activeTag`, `showAllTags`, and the existing `collapsed`/`fullWidth`/`lightbox`). The `/writing` route (no slug) renders a new blog home JSX block instead of auto-redirecting to the first post.

**Tech Stack:** React 19, TypeScript 5.9, Vite 7, i18next, React Router DOM 7, Tailwind CSS 3 (inline styles, not class-based in this component)

## Global Constraints

- Package manager: npm only — no yarn/pnpm commands.
- `npm run build` = `tsc -b && vite build` — must pass after every task; TypeScript errors fail CI.
- `npm run lint` = ESLint 9 flat config — must pass (also runs on commit via Husky pre-commit).
- Import ordering: packages → components → utils (one blank line between groups, none within).
- Commit messages must start with a capitalized infinitive verb (Add, Fix, Update…). No `Co-Authored-By` lines.
- All user-facing copy goes through `useTranslation()` — keys added to **both** `en.json` and `es.json`.
- No dark-mode variants to add; site is light-mode only (dark theme is togglable via ThemeContext).
- `TAGS_VISIBLE = 5` — first 5 tags (by `order`) shown in sidebar footer; "Show More" appears only when there are more than 5.

```ts
// Shared types + locale picker for CMS-managed content (pagescms.org writes
// these JSON files; the app reads them at build time). Each editable string is
// stored as a paired { en, es } object so translations stay side-by-side.

export type Lang = 'en' | 'es';

export interface Localized {
  en: string;
  es: string;
}

export const pickLocale = (field: Localized | undefined, lang: string): string => {
  if (!field) return '';
  const key: Lang = lang.startsWith('es') ? 'es' : 'en';
  return field[key] ?? field.en ?? '';
};

// A paired { en, es } field is only usable if it actually carries copy. Pages
// CMS can persist a half-finished entry (missing title/body) which would
// otherwise crash the reader at render time.
const isLocalized = (field: unknown): field is Localized =>
  typeof field === 'object' && field !== null && typeof (field as Localized).en === 'string';

export interface NowContent {
  lastUpdated: string;
  siversNote: Localized;
  entries: Array<{ key: string; label: Localized } & Localized>;
}

export interface WorkService {
  id: string;
  stack: string[];
  title: Localized;
  body: Localized;
}

export interface WorkContent {
  services: WorkService[];
}

export interface WritingCategory {
  id: string;
  order: number;
  label: Localized;
}

export interface WritingTag {
  id: string;
  order: number;
  label: Localized;
}

export interface WritingPost {
  slug: string;
  category: string;
  date: string;
  len: string;
  tags: string[];
  draft: boolean;
  title: Localized;
  body: Localized;
}

const categoryModules = import.meta.glob('../content/writing/categories/*.json', { eager: true });
const tagModules = import.meta.glob('../content/writing/tags/*.json', { eager: true });
const postModules = import.meta.glob('../content/writing/posts/*.json', { eager: true });

export const loadCategories = (): WritingCategory[] =>
  Object.values(categoryModules)
    .map((m) => (m as { default: WritingCategory }).default)
    .sort((a, b) => a.order - b.order);

export const loadTags = (): WritingTag[] =>
  Object.values(tagModules)
    .map((m) => (m as { default: WritingTag }).default)
    .sort((a, b) => a.order - b.order);

export const loadPosts = (): WritingPost[] =>
  Object.values(postModules)
    .map((m) => (m as { default: WritingPost }).default)
    // Drop incomplete CMS entries: a post without localized title/body has
    // nothing to render and would throw in pickLocale.
    .filter((p) => isLocalized(p.title) && isLocalized(p.body))
    .filter((p) => (import.meta.env.PROD ? !p.draft : true))
    .sort((a, b) => b.date.localeCompare(a.date));
```
