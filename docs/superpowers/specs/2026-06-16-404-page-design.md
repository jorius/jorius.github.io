# 404 Page Design

**Date:** 2026-06-16
**Status:** Approved (visual mockup v2 signed off by user)

---

## Goal

Replace the current minimal `NotFound.tsx` (small centred text inside the legacy `/*` Header/Footer layout) with a full-page 404 experience that matches the direction-b design system: `BTopBar`, a massive glitching "404" hero, and nothing below it.

---

## Visual Design

- **Full viewport height** тАФ page fills exactly 100vh: 48px topbar + remaining hero. No scroll.
- **Layout stack (top тЖТ bottom):**
  1. `BTopBar` тАФ identical to the one used on the landing page and Writing page (includes language toggle, nav links, availability status, command palette hint)
  2. Hero тАФ centred column, vertically centred in remaining height:
     - Small uppercase label: `"error ┬╖ route not found"`
     - Giant "404" with CSS glitch animation
     - "NOT FOUND." subtitle (~32тАУ36px bold)
     - One-line body copy (~13px, muted)
     - Bordered "тЖР GO HOME" link button
- **No contact section, no footer** тАФ page ends cleanly after the home link

### Glitch animation

Three-layer technique (matches the existing `Glitch` component pattern):

- **Main layer** тАФ `z-index: 2`, full opacity, skewX keyframe
- **Red channel ghost** тАФ `color: t.rgbR`, `mix-blend-mode: screen` (dark) / `multiply` (light), shifts horizontally with `clip-path` slice keyframes
- **Blue channel ghost** тАФ same as red, opposite direction/slices
- **Scanbar artifacts** тАФ two absolutely-positioned 2px horizontal bars (one red, one blue) that animate across the number during glitch bursts
- **Period:** ~2.4s (fires once per cycle, burst lasts ~12% of the period тЙИ 290ms)

The `mix-blend-mode` must switch based on theme: `screen` on dark paper (lightens), `multiply` on light paper (darkens the channel copy into the background).

### Typography & colour tokens

All values from `useBTheme()`:

| Use | Token |
|---|---|
| Background | `t.paper` |
| All text | `t.ink` |
| Label / body copy | `t.dim` |
| Borders | `t.rule` |
| Red glitch channel | `t.rgbR` |
| Blue glitch channel | `t.rgbB` |

Font: Space Mono via existing Google Fonts load in `index.html`. "404" size: `clamp(160px, 30vw, 400px)`.

## Architecture

### Route restructure (`App.tsx`)

The current `/*` catch-all wraps legacy routes in old Header/Footer and uses `<Route path="*">` inside its inner `<Routes>` to render the old `NotFound`. This makes it impossible for `NotFound` to render its own topbar.

**New structure** тАФ extract a `LegacyLayout` layout route using `<Outlet>`:

```
<Routes>
  /          тЖТ Home
  /writing   тЖТ Writing
  /writing/:slug тЖТ Writing
  /read/:slug    тЖТ Navigate тЖТ /writing
  /pgp       тЖТ Pgp
  /palette   тЖТ Palette
  (layout: LegacyLayout)
    /about   тЖТ About
    /portfolio тЖТ Portfolio
    /contact тЖТ Contact
  *          тЖТ NotFound   тЖР top-level, standalone
</Routes>
```

`LegacyLayout` is a new thin component in `src/components/LegacyLayout.tsx`:

```tsx
// Wraps the pre-redesign pages (About, Portfolio, Contact) with the
// old Header/Footer until each page gets a direction-b redesign pass.
const LegacyLayout = () => (
  <div className="min-h-screen bg-light-bg text-light-text">
    <Header />
    <main><Outlet /></main>
    <Footer />
  </div>
);
```

### `NotFound.tsx`

Standalone page component. Uses:
- `useBTheme()` for colour tokens and theme (to set blend mode)
- `useTranslation()` for all copy
- `BTopBar` тАФ renders exactly as on Home and Writing
- `ScanLines`, `DarkGrain` тАФ ambient texture (same as Writing page)
- `Glitch` component тАФ **not** used here; the glitch animation is CSS-only (inline `<style>` injected via a `<style>` tag or keyframe class) to avoid the `Glitch` component's randomised JS timer firing at different rates per ghost layer. A single CSS animation guarantees all three layers are perfectly in sync.
- `CommandPalette` + `PALETTE_SECTIONS` тАФ same as Writing, so `/` shortcut still works

The component renders no footer and no contact section.

### i18n

Update both `src/i18n/locales/en.json` and `es.json`. The existing `"notFound"` key block needs three sub-keys:

| Key | EN | ES |
|---|---|---|
| `notFound.errorLabel` | `"error ┬╖ route not found"` | `"error ┬╖ ruta no encontrada"` |
| `notFound.title` | `"NOT FOUND."` | `"NO ENCONTRADO."` |
| `notFound.body` | `"The page you're looking for doesn't exist, was moved, or the link is broken."` | `"La p├бgina que buscas no existe, fue movida o el enlace est├б roto."` |
| `notFound.home` | `"тЖР go home"` | `"тЖР ir al inicio"` |

The existing keys (`title`, `body`, `home`) are repurposed; `errorLabel` is added.
